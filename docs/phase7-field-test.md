# Phase 7: 実機フィールドテスト手順書

Phase 0〜6 はコード上の自動化で完結したが、**iPhone Safari / Android Chrome で実 GPS ハードウェアが期待通り動くか** は実機でしか確認できない。本ドキュメントは京都に行かずに**自宅近所の公園 30〜60 分散歩**で全項目を検証するための手順書。

## 検証対象

| 項目 | 期待値 | 検証方法 |
|---|---|---|
| AGPS 初期捕捉時間 | 屋外で 10 秒以内に最初の fix | 起動から `val-dist` が `0.00` 以外になるまでの秒数を計測 |
| GPS 精度 | accuracy ≤ 30m が安定して維持される | DevTools `console.log` で `pos.coords.accuracy` を観察、または緊急タブの「精度」表示 |
| Wake Lock | 「常時記録」モードで 30 分間画面が点灯し続ける | 設定→電源モードを「常時記録」にして 30 分歩く |
| 経路追従補間 | 「省電力」モードで画面オフ→再開時に補間トーストが出る | 設定を「省電力」にして 5〜10 分画面オフ→点灯し補間トースト確認 |
| 電池消費 | 1 時間あたり ≤ 25%（10000mAh モバイルバッテリーで丸 1 日カバー） | 開始時バッテリー残量と 30 分後を比較 |
| PWA インストール | 「ホーム画面に追加」できる、オフラインで起動できる | iOS Safari / Android Chrome の共有メニュー → ホームに追加 |
| Service Worker | 機内モードでアプリシェル + 訪問済みコースが起動する | 一度起動した後で機内モード ON → リロード |
| Open-Meteo | 起動時に当日予報が表示される、オフライン時は無表示 degrade | ランディング先頭の天気カード |

## 事前準備

### 1. 本番 URL の確認

GitHub Pages にデプロイ済みの URL を開く:

- https://funmatu.github.io/kyototrailplan05/

（カスタムドメインを設定する場合は `CNAME` ファイルを別 PR で追加してください。）

### 2. テストする端末

- **iPhone Safari** （iOS 16 以降推奨）
- **Android Chrome** （Android 10 以降推奨）

両方持っているなら両方で同じ手順を回し、挙動の差分を観察する。片方だけならその端末で実施。

### 3. シャドウモードを使う

自宅近所の公園で京都一周トレイルを歩くフリをするには **シャドウモード** が便利:

```
https://funmatu.github.io/kyototrailplan05/?sim=nishiyama&shadow=1
```

`shadow=1` を付けると「実 GPS で取得した座標を、選択コースの起点に平行移動して投影」するため、自宅近所で歩いた距離だけ西山コース上を進む（軌跡形状はコースと一致しない）。Wake Lock / AGPS / 電池消費の挙動はネイティブ navigator.geolocation そのままなので、本物の登山と同等の検証ができる。

`?sim=` を付けない通常 URL でも実 GPS は動くが、京都の座標から大きく外れているため `currentDistanceKm` が 0 のまま動かない。Phase 7 では shadow=1 を必ず付ける。

### 4. 前回の進捗をリセット

前回テストの残骸を消したい場合:

1. アプリを開く
2. 「歩行記録」セクションの「全リセット」ボタン
3. 確認ダイアログで OK

## 検証チェックリスト

### A. AGPS 初期捕捉

1. 屋外（建物から 5m 以上離れる、空が見える）に立つ
2. アプリを `?sim=nishiyama&shadow=1` で開く
3. ストップウォッチで時刻計測開始
4. ダッシュボードの「距離 km」が `0.00` 以外に変わったら計測終了
5. **期待**: 10 秒以内
6. **記録**: ___ 秒
7. もし 30 秒以上かかる場合: 端末の位置情報サービスが ON か、空が見えているか確認

### B. GPS 精度

1. 同じ屋外位置で 5 分静止
2. iOS Safari: 設定 → Safari → 詳細 → Web インスペクタを ON にし、Mac の Safari から接続して console.log を見る  
   Android Chrome: chrome://inspect で接続し DevTools の console を見る  
3. もしくは「緊急」タブの「精度」項目を見る
4. **期待**: accuracy ≤ 30m が 5 分間安定維持
5. **記録**: 最大値 ___ m / 最小値 ___ m

### C. Wake Lock — 常時記録モード

1. アプリを開いて「歩行記録」セクションの「電源モード・補間」→ モードを **「🔆 常時記録（Wake Lock維持）」** に変更
2. `?sim=nishiyama&shadow=1` でコース開始
3. **30 分間** 通常通りスマホを使うフリで散歩（ポケットには入れない、画面を見える状態に）
4. **期待**: 30 分間画面が消灯しない
5. **記録**: 消灯まで ___ 分（理想は 30 分以上）

> Wake Lock は OS のバッテリーセーバーモード等で強制解除される場合あり。正常時は最大 30 分以上維持できるはず。

### D. 経路追従補間 — 省電力モード

1. モードを **「🔋 省電力（画面オフOK・経路追従補間）」** に戻す
2. `?sim=nishiyama&shadow=1` で開始
3. 5 分歩く（GPS 取れている、val-dist 増える）
4. **画面を消す**（電源ボタンで sleep）
5. ポケットに入れて **5〜10 分歩く**
6. 画面を点ける
7. **期待**: トースト通知 「画面オフ中の X.X km を補間しました」
8. **記録**: 補間距離 ___ km、表示時間 ___ ms 程度

> 補間が起きない場合: 復帰位置がコースから ≥50m 離れていないか確認。許容誤差を 100m / 150m に上げて再実施。

### E. 電池消費

1. テスト開始時のバッテリー残量を記録（例: 80%）
2. 「常時記録」モードで 30 分歩く
3. テスト終了時のバッテリー残量を記録（例: 67%）
4. **計算**: (開始 - 終了) × 2 = 1 時間あたりの消費 % （例: (80-67) × 2 = 26%/h）
5. **期待**: ≤ 25%/h
6. **記録**: 開始 ___% → 終了 ___% → 1h 換算 ___%/h

> モバイルバッテリーなしで丸 1 日（10〜12 時間）の登山を耐えるには 8〜10%/h が理想。25%/h 以下なら 10000mAh のモバイルバッテリー併用で 12 時間カバー可能。

### F. PWA インストール

#### iOS Safari
1. Safari でアプリを開く
2. 共有ボタン → 「ホーム画面に追加」
3. アイコンが「京」コンパスローズになることを確認
4. ホーム画面に戻る
5. アイコンをタップ
6. **期待**: フルスクリーン（Safari の URL バーなし）でアプリ起動

#### Android Chrome
1. Chrome でアプリを開く
2. メニュー → 「アプリをインストール」または 「ホーム画面に追加」
3. アイコンが追加されることを確認
4. アイコンをタップ
5. **期待**: standalone モードで起動

### G. Service Worker — オフライン起動

1. アプリを開いて 4 コース全部を一度ずつクリック（プリキャッシュさせるため）
2. 機内モード ON
3. アプリ（PWA としてホーム画面から、もしくはブラウザのリロード）を起動
4. **期待**: 
   - ランディングが表示される
   - 4 コース全部選択できる
   - 国土地理院タイルは前回見た範囲のみ表示（未訪問エリアは灰色）

### H. Open-Meteo 当日予報

1. オンラインで起動
2. ランディング先頭にグラデーション背景の天気カード
3. **期待**: 天気アイコン + 最高/最低気温 + 降水確率 + 風速 + UV + 日出/日没 が表示
4. オフラインで再起動
5. **期待**: 天気カードは非表示（degrade）または前回キャッシュ表示

## トラブルシューティング

### val-dist が増えない
- shadow=1 を付け忘れていないか URL を確認
- 屋外に出る（屋内では GPS 弱い）
- 端末の位置情報サービス → Safari/Chrome に「常に許可」しているか確認

### 補間トーストが出ない
- 「省電力」モードか確認
- 復帰位置がコースから 50m 以内か確認（緊急タブの「精度」を確認）
- ギャップが 4 時間を超えていない、速度が 8km/h 以下を確認

### ホーム画面に追加が出ない
- iOS は 共有 → ホーム画面 にスクロールして「ホーム画面に追加」を探す
- Android はメニュー → 「アプリをインストール」（無い場合は manifest.webmanifest が読まれていない可能性、URL が https かを確認）

### Service Worker が登録されない
- DevTools → Application → Service Workers を確認
- file:// では SW が動かない（必ず https:// でアクセス）
- 古い SW がキャッシュしている場合: Application → Storage → Clear site data でリセット
